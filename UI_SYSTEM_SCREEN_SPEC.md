# Спецификация экрана «Система» (System Screen) — v1.0

## 1. Концепция

Экран заменяет стандартные «Настройки» и становится центром диагностики программно-аппаратного комплекса.
**Стиль:** Премиальный промышленный минимализм. Использование карточек ()IiotCard) с четким разделением зон ответственности.

---

## 2. Структура экрана

### Блок А: Здоровье сервисов (Service Health)

_Источник: Таблица `system_status` (через API)_

- **Collector Status**: Индикатор (Badge) + Текст.
  - `ONLINE` (Green), `DEGRADED` (Yellow), `CRITICAL` (Red).
- **Uptime**: Форматированная строка (напр. "1д 04ч 20м").
- **Last Sync**: Время последнего пакета данных (Relative time, напр. "2 сек. назад").
- **Errors**: Текст последней ошибки, если статус не ONLINE.

### Блок Б: Параметры опроса и фильтрации (Collector Config)

_Источник: Таблица `system_config` (через API)_

- **Интервал опроса (Polling)**: Значение в мс (напр. "5000 мс").
- **Мертвая зона (Deadband)**: Порог в % (напр. "0.1%").
- **Пульс данных (Heartbeat)**: Максимальный интервал записи статичных данных (напр. "600 с").
- **UI Update Rate**: Интервал обновления данных на экране (напр. "2000 мс").

### Блок В: Политика хранения (Data Lifecycle)

_Источник: Таблица `system_config` (через API)_

- **Детальные данные**: Срок хранения сырых метрик (напр. "90 дней").
- **Аналитика**: Срок хранения агрегатов (напр. "1825 дней").

### Блок Г: Настройки приложения (App Preferences)

_Нижняя часть экрана, визуально отделенная_

- **Тема оформления**: Выбор (Системная / Светлая / Темная).
- **Уведомления**: Переключатель (Push-уведомления об алармах).
- **Версия ПО**: Текущая версия сборки Mobile (напр. v1.2.0-stable).

---

## 3. Интерактивы и API

1. **Обновление данных**:
   - Блок А (Health) должен обновляться в реальном времени через SignalR или короткий опрос (5 сек).
   - При получении `config_changed` (через SignalR) экран должен мгновенно обновить значения в Блоках Б и В.
2. **Действия**:
   - Все данные на этом экране, получаемые с апи - неизменяемые (READONLY).

---

## 4. Маппинг данных (Database -> UI)

| UI Элемент       | Поле в БД                             | Тип отображения |
| :--------------- | :------------------------------------ | :-------------- |
| Collector Status | `system_status.status`                | Status Badge    |
| Last Sync        | `system_status.last_sync`             | Time Ago        |
| Polling Interval | `system_config.polling_interval_ms`   | Value + unit    |
| Deadband         | `system_config.deadband_threshold`    | Percentage      |
| UI Update Rate   | `system_config.ui_update_interval_ms` | Value + unit    |
| Raw Retention    | `system_config.raw_retention_days`    | Days            |
